ClambHook is a Surge-style proxy/VPN client spanning three repos: `clambhook` (master, Go/C daemon + Apple SwiftUI + Android + shared Skip UI), `clambercloud.com` (main, landing/download site), and `swiphtgroup.com` (main, store). macOS and Android releases go ONLY through `clambercloud.com`; GitHub is source-only and must never host installer artifacts. macOS targets >=14.0, uses its own `utun`/TUN core, supports per-process prompts and a connection map, and is gated by a durable licensing spec ($99.99 first year updates, $9.99/year renewal, 10 devices, Creem/NowPayments only) that is now fully implemented in code: Go core (`internal/license`, `internal/licensebridge`), daemon middleware (`internal/api/license.go` gates POST/PUT/DELETE on a `--license` snapshot file, fail-closed, 10s cache TTL), Apple clients (`ui/apple` via `MobileLicenseEvaluator`/`MobileLicenseRuntimeGuard`, widget intents gated), Android (`ui/android` via `LicenseManager`/`LicenseDecision.canUseApp`), and Linux (`ui/linux` via `LicenseManager`/`license_allows_use`, fail-closed). The daemon `--license` flag is wired on macOS (via `DaemonLaunchPlanner` + privileged helper) and Linux (via `DaemonSupervisor`), with the snapshot exported to a file (app group container / config dir). The hosted license backend (server, payment webhooks) lives in the swiphtgroup.com repo. Android targets 11+ (`minSdk 30`) and uses a Swift-first Skip module plus `scripts/build-android-mobile-aar.sh`. Release signing uses GPG key `EAA876B70B1832F5` and Apple Team `V6GG4HYABJ`; `make release-macos` is blocked until the `clambhook-notary` notarytool keychain profile is created manually.
